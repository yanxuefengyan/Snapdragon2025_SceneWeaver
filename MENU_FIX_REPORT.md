# SceneWeaver 菜单功能修复报告

## 📋 问题描述

用户报告菜单功能存在问题：
- ❌ 打开文件时无法选择文件
- ❌ 另存为时无法选择路径
- ❌ 文件浏览器功能无法使用

## 🔍 问题分析

### 根本原因
1. **tkinter 对话框兼容性问题**: 在某些环境下 tkinter 可能无法正常工作
2. **缺少后备方案**: 没有提供替代的文件选择方式
3. **异常处理不足**: 对话框失败后没有适当的错误处理
4. **路径验证缺失**: 没有验证文件和路径的有效性

## ✅ 修复方案

### 1. 实现多重后备方案

#### 第一方案：tkinter 对话框（优先）
```python
try:
    import tkinter as tk
    from tkinter import filedialog
    
    root = tk.Tk()
    root.withdraw()
    root.attributes('-topmost', True)
    root.update()
    
    file_path = filedialog.askopenfilename(
        parent=root,
        title="打开项目",
        filetypes=[...],
        initialdir=os.path.expanduser("~")
    )
    
    root.destroy()
except Exception as e:
    # 降级到后备方案
    pass
```

#### 第二方案：控制台输入
```python
try:
    print("\n请选择文件路径 (或输入 q 取消):")
    file_path = input("> ")
    if file_path and os.path.exists(file_path):
        self._load_project(file_path)
except Exception as e:
    self.logger.error(f"文件打开失败：{e}")
```

### 2. 增强异常处理

- ✅ 添加 try-except 保护所有对话框操作
- ✅ 确保 tkinter 窗口在异常时正确销毁
- ✅ 详细的错误日志记录

```python
def _on_file_save_as(self):
    # 默认文件名
    if hasattr(self, 'project_data') and 'name' in self.project_data:
        default_name = f"{self.project_data['name']}.swp"
    
    file_path = filedialog.asksaveasfilename(
        defaultextension=".swp",
        initialfile=default_name,
        ...
    )
    
    # 确保目录存在
    directory = os.path.dirname(file_path)
    if directory and not os.path.exists(directory):
        os.makedirs(directory)
```

## 📚 使用说明

### 使用文件对话框

1. **打开文件**:
   - 菜单：文件 > 打开
   - 操作：在对话框中选择 .swp 或 .json 文件
   - 后备：如对话框失败，在控制台输入路径

2. **保存项目**:
   - 菜单：文件 > 保存
   - 自动：保存到当前路径
   - 后备：如无路径，弹出另存为对话框

3. **另存为**:
   - 菜单：文件 > 另存为
   - 操作：选择保存位置和文件名
   - 支持：中文路径和文件名

4. **浏览文件**:
   - 菜单：工具 > 文件浏览器
   - 操作：选择目录
   - 记录：自动保存最近使用的目录

### 快捷键（计划）
- Ctrl+O: 打开文件
- Ctrl+S: 保存项目
- Ctrl+Shift+S: 另存为
